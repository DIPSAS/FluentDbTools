WithChangeLog(IAlterTableAddColumnOrAlterColumnSyntax, ChangeLogContext)

ChangeLog activation for Alter.Table(..) syntax
-------------------------------------------------
It will be possible to add custom logging on all table operations.
The interface ICustomMigrationProcessor<T> must be implemented and registered in the IoC container (IServiceCollection )
Method Process(IChangeLogTabledExpression)
can be used to implementing your preferred custom logging

See the example of a implementation of ICustomMigrationProcessor<T>:
https://github.com/DIPSAS/FluentDbTools/blob/master/src/FluentDbTools/Tests/Test.FluentDbTools.Migration/TestOracleCustomMigrationProcessor.cs

Can be registered like this:

IServiceCollection.AddSingleton<ICustomMigrationProcessor<OracleProcessor>,TestOracleCustomMigrationProcessor>();

WithForeignKeyColumn(ICreateTableWithColumnSyntax, String, MigrationModel, String, String)

Create a foreign columnName to the table primaryTableName
------------------------------------------------------------------
If columnName is specified, the new column-name will follow that value
If columnName is unspecified, the new column-name will be computed

If primaryColumnName is specified, the primary-table-column will follow that value
If primaryColumnName is unspecified, the primary-table-column will be computed

i.e:
columnName is unspecified and primaryTableName is "Parent" => column-name will be computed to "ParentId"

primaryColumnName is unspecified => the primary-table-column will be "Id"
